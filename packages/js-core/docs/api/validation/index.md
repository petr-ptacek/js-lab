# validation

Utilities in the **validation** category.

<ul>
<li><a href="./isArray">isArray</a> — Checks whether the given value is an array.</li>
<li><a href="./isBigint">isBigint</a> — Checks whether the given value is a bigint.</li>
<li><a href="./isBoolean">isBoolean</a> — Checks whether the given value is a boolean.</li>
<li><a href="./isDate">isDate</a> — Checks whether the given value is a Date instance.</li>
<li><a href="./isFiniteNumber">isFiniteNumber</a> — Checks whether the given value is a finite number.</li>
<li><a href="./isFunction">isFunction</a> — Checks whether the given value is a function.</li>
<li><a href="./isNaNValue">isNaNValue</a> — Checks whether the given value is NaN (Not-a-Number).</li>
<li><a href="./isNull">isNull</a> — Checks whether the given value is null.</li>
<li><a href="./isNullable">isNullable</a> — Checks whether the given value is null or undefined.</li>
<li><a href="./isNumber">isNumber</a> — Checks whether the given value is a number.</li>
<li><a href="./isObject">isObject</a> — Checks whether the given value is an object (including arrays, but excluding null).</li>
<li><a href="./isPlainObject">isPlainObject</a> — Checks whether the given value is a plain object.</li>
<li><a href="./isPrimitive">isPrimitive</a> — Checks whether the given value is a JavaScript primitive.</li>
<li><a href="./isString">isString</a> — Checks whether the given value is a string.</li>
<li><a href="./isSymbol">isSymbol</a> — Checks whether the given value is a symbol.</li>
<li><a href="./isUndefined">isUndefined</a> — Checks whether the given value is undefined.</li>
<li><a href="./isValidDate">isValidDate</a> — Checks whether the given value is a valid Date instance.</li>
</ul>
